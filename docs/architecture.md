# ClawMe 架构要点

## 整体数据流

```
OpenClaw（或其它 Agent）  →  推送/后端（带 token）  →  ClawMe 客户端（浏览器 / 手机）
                                                              ↓
                                                    校验 token → 展示指令 → 用户执行/半自动执行
```

## 指令类型（示意）

| 类型 | 说明 | 主要执行端 |
|------|------|-------------|
| `remind` | 仅提醒 + 文案 | 手机 |
| `open_url` | 打开 URL | 浏览器 / 手机 |
| `fill_form` | 在页面填表 | 浏览器插件 |
| `click` | 点击页面元素 | 浏览器插件 |
| `extract` | 从页面抓取内容回传 | 浏览器插件 |
| `send_sms` | 发短信（预填内容） | 手机 |
| `run_shortcut` | 运行苹果快捷方式 | 手机 |
| `open_wechat` | 打开微信到某聊天 | 手机（需 URL scheme / 手动最后一步） |

## 谁先做：浏览器 vs 手机

- **手机先行**：适合「提醒 + 发消息 / 快捷方式」故事，闭环更直观。
- **浏览器先行**：演示「填表、点按钮」快，和现有扩展技术栈接近。
- 建议：先定**指令协议 + 一个端**，再补第二端；两端共用同一套协议与 token。

## 与 Telegram 的对比

- Telegram：传**内容**（链接、文字）→ 用户手动操作。
- ClawMe：传**操作**（结构化指令）→ 客户端在设备上执行（或半自动执行）。
