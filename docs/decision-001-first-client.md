# 决策记录 001：第一端选型

**日期**：2025-02  
**状态**：已采纳  

## 背景

MVP 需要「至少一端」可执行指令。可选：手机先行（提醒+发短信）或浏览器先行（填表/点按钮、易开源）。

## 决策

**第一端选型：浏览器插件先行。**

## 理由

- 技术栈通用（Web 扩展），便于开源贡献与社区参与。
- 不需真机即可开发、演示（open_url、remind 即可玩起来）。
- 与 roadmap 中「先玩起来、再补手机」路径一致；发短信/手机端作为第二阶段。

## 后续

- 后端与协议同时支持 `target: "browser"` 与 `target: "phone"`，便于后续补手机端。
- 浏览器插件首版支持：`remind`、`open_url`；执行结果上报与可选回传 OpenClaw。
